# Мультиклиент: несколько клубов на одном сервере

Один процесс обслуживает несколько клубов, у каждого — собственные
Telegram-бот и/или сообщество ВК. Данные изолированы по клубам.

## Панель оператора (проще, чем curl)

Вместо ручных запросов ниже — веб-форма: `/admin/platform/login`, вход по
`ADMIN_API_TOKEN`. Дашборд со списком клиентов, кнопка «Добавить клиента»
(создаёт клуб + подключает боты + hot-reload за одну отправку формы),
быстрое продление оплаты прямо из таблицы. См. app/admin/platform.py.

## Как подключить нового клиента через API (на вашем сервере)

1. Клиент создаёт своего бота у @BotFather и/или сообщество ВК,
   передаёт вам токены.
2. Создайте клуб:  POST /api/tenants  {"name": "Клуб клиента"}
   (заголовок x-admin-token).
3. Задайте токены клуба:
   PATCH /api/tenants/{id}/tokens  {"tg_token": "...", "vk_token": "..."}
4. Привяжите тренера клиента:
   PATCH /api/tenants/{id}/chat  {"admin_tg_id": ..., "admin_vk_id": ...}
5. Боты клиента поднимутся автоматически сразу после шага 3
   (hot-reload; в ответе будет "reloaded": true). Если по какой-то
   причине не поднялись — перезапустите сервис (Railway → Restart).
6. Тренер клиента пишет своему боту /start — видит меню управления.

## Как это работает

- Токены хранятся в базе (tenants.tg_token / tenants.vk_token).
- При старте процесс поднимает бота по каждому токену и поллит все сразу.
- Событие, пришедшее клиентскому боту, автоматически относится к его
  клубу (по токену) — участники и тренеры разных клубов не пересекаются.
- Исходящие (напоминания, рассылки, публикации) уходят через бота
  того клуба, которому принадлежит адресат.
- Бот из переменных окружения (TG_TOKEN/VK_TOKEN) работает как раньше —
  это «клуб №1» владельца платформы.

## Оплата клубов (SaaS)

- PATCH /api/tenants/{id}/billing  {"paid_until": "2026-08-01"}
  — до какой даты клуб оплачен. Пустая строка — без ограничений.
- После даты боты и веб-страница клуба отвечают
  «работа приостановлена»; напоминания и автосоздание тренировок
  не выполняются. Владелец платформы получает список истекающих
  клубов в Telegram за 3 дня до даты.

## Ограничения

- ВК-бот клиента должен иметь включённый Long Poll API в настройках
  сообщества (Сообщения → Настройки для бота + API).
