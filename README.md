#  Бот записи на тренировки

[![CI](https://github.com/Hydwild/sports-training-bot/actions/workflows/ci.yml/badge.svg)](https://github.com/Hydwild/sports-training-bot/actions/workflows/ci.yml)

Бот для Telegram и ВКонтакте, который ведёт запись на тренировки: участники
записываются сами, при заполнении мест встают в очередь, получают напоминания.
Есть две версии — **Lite** (для одного тренера) и **Pro** (для клуба).

---

## Какую версию выбрать

### 🟢 Lite — для одного тренера
Простая версия, всё необходимое для ведения своих тренировок:
- **Запись** на тренировку через бота (кнопкой).
- **Очередь** — когда мест нет, человек встаёт в очередь и автоматически
  занимает место, если кто-то отменил запись.
- **Напоминания** — участникам приходит напоминание перед тренировкой.
- **Посещаемость** — после тренировки отмечаете, кто реально пришёл.

Работает на лёгкой базе (SQLite), не требует отдельного сервера базы данных.

### 🔵 Pro — для клуба
Всё из Lite, плюс:
- **Статистика** — рейтинг посещаемости, графики, профиль участника.
- **Оплаты** — участник платит онлайн (ЮKassa), после оплаты автоматически
  отмечается как оплативший.
- **Несколько групп** — отдельные клубы и группы внутри клуба (дети/взрослые).
- **Веб-админка** — панель управления в браузере с ролями
  (владелец / тренер / ассистент), настройками и экспортом в Excel/PDF.
- **Должники** — список не оплативших и рассылка напоминаний.

Работает на PostgreSQL (поднимается автоматически).

## Быстрое развёртывание на сервере (одной командой)

Нужен только сервер с Linux (например, обычный VPS). Всё остальное скрипт
сделает сам.

```bash
# 1. Скопируйте проект на сервер и зайдите в папку
cd badminton_bot_v2

# 2. Запустите развёртывание нужной версии:
./deploy.sh lite     # для одного тренера
#   или
./deploy.sh pro      # для клуба
```

Скрипт сам:
1. поставит Docker, если его нет;
2. создаст файл настроек `.env` (для Pro сразу сгенерирует секретные ключи);
3. попросит вписать **токен бота** (получить — у @BotFather в Telegram);
4. соберёт и запустит всё в контейнерах;
5. покажет адрес и статус.

После запуска бот работает. Проверить: откройте `http://АДРЕС_СЕРВЕРА:8000/health`.
Для Pro админка — `http://АДРЕС_СЕРВЕРА:8000/admin`.

**Полезные команды после запуска:**
```bash
docker compose -f docker-compose.pro.yml logs -f app   # смотреть логи
docker compose -f docker-compose.pro.yml down          # остановить
docker compose -f docker-compose.pro.yml up -d          # запустить снова
```
(для Lite — замените `pro` на `lite`)

## Что вписать в настройки (.env)

Минимум для запуска:
- **TG_TOKEN** — токен Telegram-бота от @BotFather. Обязательно.

Для ВКонтакте (необязательно): **VK_TOKEN** — ключ сообщества.

Для Pro дополнительно:
- **JWT_SECRET**, **ADMIN_API_TOKEN** — секреты (скрипт генерирует сам).
- **YOOKASSA_SHOP_ID**, **YOOKASSA_SECRET_KEY** — для приёма оплат
  (из личного кабинета ЮKassa).
- **TG_BOT_USERNAME** — имя бота для входа в админку через Telegram.
- **PHONE_ENC_KEY** — выделенный ключ шифрования телефонов клиентов.
  Хранится **отдельно от резервных копий**: копия зашифрована им же, и
  ключ внутри неё сделал бы шифрование бессмысленным. Пусто — телефоны
  шифруются историческим ключом, выведенным из `JWT_SECRET`.
- **PHONE_KEYRING** — связка прежних ключей (`jwt:<старый JWT_SECRET>`) на
  время перехода.
  Версии ключей неизменяемы: два разных секрета под одной версией (включая
  дубль в одной переменной и конфликт `PHONE_ENC_KEY` с `PHONE_KEYS`) —
  приложение не стартует. Замена ключа = новая версия, а не новый секрет
  под старой меткой; перекрыть можно только историческую `jwt` при ротации
  `JWT_SECRET`. Подробности — в
  [DISASTER_RECOVERY.md](DISASTER_RECOVERY.md#именованные-версии-ключей-и-строгий-реестр).
- **PUBLIC_BASE_URL** — публичный адрес (`https://...`). Из него строятся
  абсолютные ссылки: QR-код страницы записи и персональная ссылка
  управления, которую администратор выдаёт клиенту. Берётся только отсюда,
  а не из заголовка `Host` запроса.

- **TRUSTED_PROXIES** — кому доверять заголовок `X-Forwarded-For`
  (uvicorn `--forwarded-allow-ips`). Пусто — заголовок игнорируется:
  подделать свой адрес нельзя, но за прокси все посетители выглядят
  одинаково и лимит запросов становится общим.
  На **Railway/Render** переменная **обязательна**: приложение определяет
  такую среду и без `TRUSTED_PROXIES` не стартует (это ровно тот боевой
  дефект, что чинили). Значение `TRUSTED_PROXIES=*` допустимо **только**
  когда приложение доступно исключительно через доверенный прокси
  платформы, а прямой доступ к контейнеру закрыт. Диагностический флаг
  `proxy_headers_configured` виден в `/health` (без раскрытия адресов).
  Задать переменную в Railway — **ручное действие владельца**.

`/health` отдаёт ещё и `keys_ok`. Это не только проверка конфигурации: она
расшифровывает реальные строки базы и заново считает по ним поисковый
индекс. Подменённый ключ под формально существующей версией (например
`jwt` после ротации `JWT_SECRET` без сохранения старого секрета) даёт
`keys_ok: false` и HTTP `503` — платформа видит такой контейнер как
нездоровый, а не как «ok». Ни секретов, ни телефонов, ни адресов в ответе
нет.

Там же есть `rss_mb` — текущая память процесса. Она нужна для контроля
счёта: Railway тарифицирует **среднюю** память (≈$10 за ГБ в месяц, то есть
каждые 100 МБ ≈ $1/мес), и без возможности посмотреть RSS прямо в проде
любая оптимизация памяти остаётся гаданием. Держите значение в поле зрения
после деплоя и особенно после суточного бэкапа: если RSS не возвращается к
исходному, значит что-то удерживает память.

⚠️ На **существующей** базе `PHONE_ENC_KEY` нельзя просто «включить»:
клиенты перестанут находиться по номеру и заведутся дублями. Порядок
перехода и команда `scripts/migrate_phone_keys.py` описаны в
[DISASTER_RECOVERY.md](DISASTER_RECOVERY.md#01-ключи-без-которых-бэкап-бесполезен).

## Как пользоваться ботом

**Запись в боте идёт воронкой:** кнопка меню открывает выбор дня, затем
свободное время этого дня (с числом мест или пометкой «очередь»), и только
потом карточку, где виден специалист. Плоский список годился, пока слотов
пять; у салона или репетитора окон на неделю десятки. Команда `/list`
осталась плоским списком — она удобна админу и в групповых чатах.

**Участники** (в чате клуба или в личке с ботом):
- `/list` — список тренировок, запись и отмена кнопками.
- `/profile` — личная статистика (в Pro).
- кнопка «Записать гостя» — записать того, кто сейчас не в сети; тренер
  потом подтвердит.

**Тренер / админ:**
- `/new` — создать тренировку (пошагово: название, дата, место,
  длительность, лимит, в Pro — цена).
- `/attend` — отметить, кто пришёл и кто оплатил.
- `/guests` — подтвердить или отклонить записанных гостей.
- `/drafts` — открыть запись на отложенную тренировку.
- `/cancel`, `/setmax`, `/broadcast` — отменить, изменить лимит, рассылка.
- `/site` — ссылка на публичную страницу записи клуба (её же можно
  дать клиентам или напечатать в QR-коде).
- (Pro) `/stats`, `/debtors`, `/export` — статистика, должники, выгрузка списка.

## Веб-админка (только Pro)

Откройте `/admin`, войдите через Telegram. Доступ — у тех, кому назначена роль:
- **Владелец (owner)** — всё, включая настройки и роли.
- **Тренер (coach)** — тренировки, отметки, экспорт, настройки.
- **Ассистент (assistant)** — только отметки явки/оплаты и просмотр.

Персональную ссылку управления клиенту (кнопка в карточке тренировки)
выдают **владелец и тренер** — ассистент её не видит и выдать не может.
Ссылка одноразовая, показывается один раз и только в этом ответе; выдавать
её можно лишь после того, как администратор сам сверил личность клиента.

На странице **⚙️ Настройки** (владелец и тренер) настраиваются:
брендинг (название, цвет, логотип), таймеры напоминаний, напоминание о
неподтверждённых гостях, авто-снятие неподтверждённых гостей, уведомление об
открытии записи, окно отмены (за сколько минут до тренировки запретить отписку).

## Адрес страницы записи

У каждого клуба в карточке есть три поля адреса, по возрастанию приоритета:

1. по умолчанию — наша `/club/<id>`;
2. **короткий адрес** (`slug`) — открывается как `/c/salon-hortensia`.
   Ссылку печатают в QR и диктуют по телефону, и «слэш клуб слэш три» для
   этого не годится. Старая `/club/<id>` при этом продолжает работать —
   уже сделанные ссылки и QR-коды ломать нельзя;
3. **своя ссылка на сайт** — если у клиента собственный сайт или домен.
   Она перекрывает всё остальное.

Выбранный адрес уходит в QR-код, кнопку бота, `/site` и список клиентов —
везде один и тот же, иначе распечатанный QR вёл бы не туда.

## Витрина направлений на лендинге

Если демо-клубов больше одного, `/promo` показывает карточку на каждое
направление: ссылку на ЕГО бота (поле «@username бота клуба») и на его
страницу записи. Одного демо мало — владелец салона, увидев спортивную
секцию, решает, что платформа не про него.

## Своя ссылка на страницу записи

В карточке клуба есть поле **«Своя ссылка на страницу записи»**. Пусто —
используется наша `/club/<id>`. Если задать, именно этот адрес уходит в
QR-код, в кнопку бота, в команду `/site` и в список клиентов: у клуба
может быть свой сайт или домен, и распечатанный QR должен вести туда.

Принимается только `https` и только внешний хост. Адрес попадает в
inline-кнопку Telegram и в QR-код: `javascript:` там означал бы XSS у
каждого, кто нажмёт, а внутренний адрес уводил бы посетителя во
внутреннюю сеть.

## Выключение и удаление клуба

В карточке клуба (`/admin/platform/<id>/edit`) есть два разных действия:

- **Выключить** — снять галочку «Клуб включён». Бот и страница записи
  перестают отвечать, но все данные остаются. Обратимо, включается тем же
  переключателем. Почти всегда нужно именно это.
- **Удалить навсегда** — блок «Опасная зона» внизу страницы. Вместе с
  клубом каскадом исчезают его занятия, записи, участники, платежи,
  отзывы, мастера и веб-клиенты с телефонами. Восстановить можно только из
  резервной копии.

Чтобы удалить, нужно **напечатать название клуба точно** — кнопки
недостаточно. Это защита от промаха: кнопку рядом с «изменить» рано или
поздно нажимают случайно, а напечатать чужое название по ошибке нельзя.
Факт удаления пишется в журнал (кто и что), токены и персональные данные
в запись не попадают.

## Направления бизнеса (вертикали)

Бот и страница записи говорят на языке клиента. Направление выбирается при
создании клуба и меняется в панели оператора:

- **Спорт и тренировки** — клубы, секции, персональные тренировки;
- **Салон красоты** — барберы, маникюр, волосы, депиляция;
- **Репетиторы и обучение** — языковые школы, подготовка к экзаменам, кружки.

От направления зависят кнопки меню, слово для специалиста (тренер / мастер
/ преподаватель) и тексты публичной страницы. Существующие клубы без
указанного направления работают как «спорт» — ничего не меняется.

**Из демо-бота можно перейти на сайт.** Кнопка со ссылкой на
страницу записи показывается сразу при выборе роли и ещё раз после
него: половина ценности продукта — в том, как клуб выглядит у
КЛИЕНТА в браузере (витрина, фото, мастера, выбор времени), и без
явной ссылки эту часть демо просто не находят. Кнопка появляется
только при заданном `PUBLIC_BASE_URL` — Telegram принимает в кнопке
лишь абсолютный http(s)-адрес.

**Демо-клубы наполняются по направлению.** Клуб с флагом «демо»
пересобирается каждую ночь примерами своей вертикали: у салона — стрижки и
маникюр с индивидуальной записью, у репетитора — и занятия один на один, и
группы. Витрина должна говорить на языке того, кто пришёл её смотреть.
Чтобы завести демо для направления: создайте клуб, выберите направление,
включите «демо» и подключите ему отдельного бота.

## Консоль площадки в Telegram (только владелец)

Если задан **PLATFORM_OWNER_TG_ID**, в личке с платформенным ботом
появляются команды владельца — быстрый просмотр с телефона, не заменяющий
веб-панель `/admin/platform`:

- `/clients` — все клубы: подключённые площадки и сколько осталось аренды
  (горящие показываются первыми);
- `/client <id>` — карточка клуба: занятия и записи за 7 дней, участники,
  предстоящие занятия;
- `/load` — память процесса, очередь уведомлений, ошибки ответов;
- `/logs` — журнал ошибок **зашифрованным** файлом;
- `/webhooks` — почему бот клуба молчит: сверяет адрес, прописанный
  в Telegram, с текущим. Смена домена (`PUBLIC_BASE_URL`) не
  перепрописывает вебхуки автоматически — Telegram продолжает слать
  на старый адрес, и бот замолкает без единой записи в логе, потому
  что запрос до нас не доходит;
- `/newclub <название>` — завести клуб;
- `/extend <id> <дней>` — продлить аренду.

Команды видит только владелец и только в личном чате: в группе их нет, а
для постороннего они попросту не существуют — бот не подаёт вида, что
такие команды есть.

⚠️ **Токены ботов в переписку не отправляются и не принимаются.** История
чата Telegram вечна, синхронизируется на все устройства и попадает в
копии самого мессенджера, а токен — это полный контроль над ботом клиента.
Поэтому `/newclub` только заводит клуб, а токен вводится в веб-панели.
Журнал ошибок по той же причине шифруется ключом резервных копий: в нём
имена, id участников и названия клубов.

## Для разработчиков

Запуск локально без Docker:
```bash
pip install -r requirements.txt
cp .env.lite.example .env       # или .env.pro.example
uvicorn app.main:app --reload
```

Тесты:
```bash
pytest -q
```

Структура и архитектура подробно описаны в комментариях кода. Кратко:
`app/core` (конфиг, безопасность, редакции), `app/models` (БД),
`app/repositories` (доступ к данным с изоляцией клубов), `app/services`
(логика: запись, оплаты, экспорт, фоновые задачи), `app/bots` (Telegram/VK),
`app/admin` (веб-панель), `app/api` (REST). Миграции — Alembic.

Разница версий задаётся одной переменной `EDITION=lite|pro` — код общий,
Pro-функции включаются флагами (`app/core/features.py`).
