# Sports Training Platform — мультиклубная система записи на тренировки

Двухплатформенный бот (Telegram + ВКонтакте) для управления записью на спортивные тренировки, с общим ядром, мультитенантностью, платежами и веб-админкой. Один сервис обслуживает несколько независимых клубов: участники записываются из привычного мессенджера, тренер управляет всем из бота или браузера.

> Проект развёрнут в проде (Railway + PostgreSQL) и используется на реальных данных.

---

## Чем интересен проект

Это не учебный пример, а рабочий продукт. Технически примечательны:

- **Мультитенантность с изоляцией данных.** Один инстанс обслуживает много клубов; каждый запрос к БД фильтруется по `tenant_id`, данные клубов не пересекаются.
- **Два бота на разных фреймворках, одно ядро.** Telegram (`aiogram`) и ВКонтакте (`vkbottle`) — разные API и модели событий, но общая бизнес-логика, репозитории и сервисы. Запись из ВК видна в Telegram и наоборот.
- **Защита от гонок на уровне БД.** Одновременная запись на последнее место, параллельные отмены и изменение лимита сериализуются через `SELECT ... FOR UPDATE` — без превышения вместимости и рассинхрона очереди.
- **Безопасность не для галочки.** JWT (HS256), проверка подписи Telegram Login Widget через `hmac`, timing-safe сравнение служебного токена, отсутствие секретов в коде.
- **Покрытие тестами и чистый линтер.** 46 автотестов (изоляция клубов, роли, записи, гости, платежи, экспорт, веб-запись, расписания), проходящий `ruff`.

## Возможности

**Для участника**
- Просмотр ближайших тренировок карточками с прогресс-баром занятости
- Запись / отмена в одно нажатие, автоматическая очередь при заполнении
- «Мои записи» — где записан и на каком месте, отмена оттуда же
- Личный профиль и статистика посещений
- Живое обновление карточки после действия

**Для тренера (админа)**
- Создание тренировки пошагово, кнопками: дата, время, место, длительность, цена, лимит
- Редактирование любого поля после публикации с авто-уведомлением записанных
- Отмена тренировки с подтверждением и оповещением участников
- Отметка явки и оплаты, учёт должников и напоминания им
- Запись «гостей» (людей без аккаунта), подтверждение/отклонение
- Приватные подписи участников (видит только тренер)
- Массовая рассылка всем подписчикам обеих платформ
- Публикация анонса на стену сообщества ВК
- Черновики и отложенная публикация

**Платформа**
- Регулярное расписание: тренировки создаются автоматически каждую неделю
- Публичная веб-страница записи (без Telegram/ВК): имя + телефон, персональная ссылка отмены, QR-код для печати
- Конфигуратор в админке: собирает готовую сборку бота под нового клиента (код + настройки + преднастроенная база)
- Резервные копии базы: кнопка в админке + автобэкап ежедневно
- Алерты владельцу в Telegram при сбоях фоновых задач
- Мультиклиент: несколько клубов с собственными ботами на одном сервере, hot-reload токенов без рестарта, изоляция данных
- Онлайн-оплата (YooKassa)
- Экспорт списков в CSV / Excel / PDF, графики посещаемости
- Веб-админка (FastAPI + Jinja) со входом через Telegram Login Widget и ролями
- Редакции продукта (lite / pro) через фиче-флаги

## Архитектура

```
app/
├── main.py            — точка входа, старт ботов и API, авто-миграции
├── core/              — конфиг, безопасность (JWT, подписи), фиче-флаги, логи
├── models/            — SQLAlchemy-модели (клубы, тренировки, записи, платежи…)
├── db/                — async engine и фабрика сессий
├── repositories/      — доступ к данным; авто-фильтрация по tenant_id
├── services/          — бизнес-логика (записи, очередь, платежи, экспорт, рассылки)
├── bots/              — Telegram- и VK-боты + общие представления карточек
├── api/               — REST-эндпойнты управления клубами
└── admin/             — веб-панель (Jinja-шаблоны)
```

Принцип: боты и API — тонкие слои над сервисами; вся логика в `services`, весь доступ к данным — в `repositories` с обязательной привязкой к клубу. Это даёт изоляцию тенантов и упрощает добавление новых платформ.

## Запуск локально

```bash
# 1. Зависимости
pip install -r requirements.txt

# 2. Переменные окружения (пример)
export DATABASE_URL="postgresql+asyncpg://user:pass@localhost/db"
export TG_TOKEN="<telegram-bot-token>"
export VK_TOKEN="<vk-community-token>"      # опционально
export JWT_SECRET="<случайная-строка>"
export ADMIN_API_TOKEN="<токен-для-API>"

# 3. Старт
uvicorn app.main:app --host 0.0.0.0 --port 8080
```

Таблицы создаются автоматически при первом старте; недостающие колонки до-мигрируются.
